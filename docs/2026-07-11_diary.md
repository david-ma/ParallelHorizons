# Gallery repository review — 2026-07-11

First-pass review of the **gallery** repo: a Thalia-hosted 3D WebGL art gallery derived from OwlSketch’s original Met Museum scraper concept, now maintained as David Ma’s exhibition digital twin and a teaching example for CDS2704.

---

## Executive summary

The project is in a **solid mid-transition state**. The Thalia conversion, TypeScript modularisation, Three.js r183 upgrade, Rapier physics, floorplan editor, and JSON-driven layout pipeline are largely done and working. What remains is mostly **productisation**: schema contract, async loading, auth/uploads, persistence, placards, and cleaning up legacy assets and docs.

Typecheck passes (`bun run typecheck`). Uncommitted work in the working tree: `package.json` (Thalia dep + scripts) and `bun.lock`.


## Architecture (viewer)

The viewer is well split compared to the original monolithic `main.js`:

```
main.ts          → boot, pointer lock, render loop, physics vs legacy branch
layout.ts        → load floorplan JSON, build walls + artworks
gallery.ts       → hardcoded default scene (30 paintings + one spotlight rig)
artwork.ts       → frames and moulding
spotlight.ts     → GLB fixture + dev sliders + export JSON
movement.ts      → WASD / gamepad, camera-relative velocity
physics.ts       → Rapier floor, walls, player body
types.ts         → Gal, FloorplanBlob
```

**Movement:** WASD and analog stick use camera-relative forward/right on the XZ plane (`movement.ts`). Correct use of `dir × up` for right vector is documented in AGENTS.md.

**Physics:** Rapier initialises async; colliders are built from `wallGroup` meshes after scene creation. Active only when pointer lock is on (`usePhysics = physicsWorld && playerBody && !screensaver`). Screensaver mode keeps legacy manual AABB sliding against walls inside `main.ts`’s render loop.

**Layout modes:**

1. If `GET /gallery-floorplan.json` succeeds → `buildSceneFromFloorplan`
2. Else → `buildDefaultGallery` (hardcoded room, 30 images, featured spotlight)

There is **no `public/gallery-floorplan.json` in the repo** right now (latest commit: “Go back to default floorplan”), so `/view` always uses the default layout unless a file is added manually.

## Architecture (create page)

`/create` is a self-contained floorplan editor:

- 5×5 grid; click cells to activate/deactivate
- Drag photos onto wall bands (north/east/south/west)
- **One photo per wall** in the editor UI
- Export / import / download JSON with `version: 2`
- Charcoal Studio dark palette, scrollable photo sidebar
- D3 v7 loaded from CDN; logic in `gallery_creation.ts`

Mock catalog: 12 photos pointing at `/img/Artworks/0.jpg` … `11.jpg`.

## JSON schema — current contract (informal)

Editor exports (`gallery_creation.ts`):

```json
{
  "version": 2,
  "grid": { "rows": 5, "cols": 5 },
  "activeCells": ["2,2"],
  "placements": {
    "2,2": { "north": "photo-0", "east": "", "south": "", "west": "" }
  },
  "photoCatalog": [{ "id": "photo-0", "title": "Photo 1", "src": "/img/Artworks/0.jpg" }]
}
```

Viewer consumes (`types.ts` / `layout.ts`):

- Same shape, but **viewer accepts `string | string[]` per wall** and can place **multiple artworks per wall** with horizontal spacing
- Backward compat: whole-cell string or legacy shapes normalised on import in the editor
- `photoCatalog[].title` is typed but **not rendered** (placards not implemented)

**Gap:** AGENTS.md changelog mentions “multi-photo per cell” on the viewer side, but the editor was simplified to one photo per wall. The schema is not formally documented or version-validated beyond the editor’s `version: 2` field.

## Strengths

1. **Clear module boundaries** — easy to navigate for agents and humans; AGENTS.md is an unusually good handoff doc.
2. **Modern stack** — Three.js 0.183.2 via npm, strict TypeScript, Rapier for collision.
3. **Dual layout path** — JSON floorplan + sensible fallback keeps `/view` usable without setup.
4. **Dev ergonomics** — spotlight tuning sliders (localhost only), JSON export, floorplan editor.
5. **Art presentation** — JSON mode gets black frame + white moulding; overlap bug was fixed.
6. **Pointer-lock UX** — Esc / blur / visibility release to menu; re-entry via PLAY overlay.

## Issues and technical debt

### Documentation drift

- **README.md** todo list is stale (still lists “refactor main.js to TypeScript” and “Three.js r183” as future work — both done).
- README quickstart is accurate; AGENTS.md is more current than README.

### Loading and lifecycle

- **`loadFloorplan` uses synchronous XHR** (`layout.ts`) — blocks the main thread; AGENTS.md already flags async load + loading state as near-term work.
- Textures and GLB load via `TextureLoader` / `GLTFLoader` without central loading state or error UI.

### Feature gaps (viewer vs default)

- **Spotlights only in default gallery** — JSON-built scenes get ambient + hemisphere light only; no per-artwork rigs.
- **No placards** — title/artist/year in schema but not shown in 3D.
- **Raycast / painting selection** — legacy path still raycasts in render loop, but selection UI from original OwlSketch alpha may be incomplete in Thalia templates.

### Code quality notes

- **`main.ts` is still the largest file (375 lines)** — render loop mixes physics step, legacy collision, wall colour reset, and framerate; candidate for extraction.
- **`Gal` uses `(window as any)` / `(g as any)` in places** — workable but weakens strict typing.
- **`gallery_creation.ts` declares `d3: any`** — fine for a dev tool, could use `@types/d3` if tightened.
- **jQuery** on `gallery.hbs` only for `#gamepadtest` dev toggle — could be vanilla JS.

### Legacy / duplicate assets

Still present under `public/` from pre-Thalia era:

- `public/index.html`, `public/js/three.js`, `three.min.js`, `three.module.js`
- `public/js/PointerLockControls.js`, `newPointerLockControls.js`
- `public/js/GLTFLoader.js` (viewer now imports from `three` npm package)
- `system_programs/python/scraper.py`, `newscraper.py`, `cpp/minAlgo.cpp`

These are likely **unused** in the Thalia flow but add noise and confusion for new contributors.

### Testing and CI

- No test script (removed placeholder `test` echo in uncommitted `package.json`).
- No automated checks beyond manual typecheck.

## Session notes — 2026-07-11 (afternoon)

### Triplex decision

[Triplex](https://triplex.dev/) targets **React Three Fiber** visual editing. Our stack is vanilla Three.js + Thalia + JSON floorplans. **Not adopting Triplex** without a full R3F rewrite. Better to extend `/create` and the JSON schema.
